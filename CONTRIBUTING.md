# Contributing to ComfyFlow Insight

Thank you for your interest in contributing to ComfyFlow Insight.

We welcome contributions that improve workflow analysis, workflow intelligence, user experience, performance, documentation, and developer tooling.

---

# Ways to Contribute

Contributions may include:

* Bug fixes
* Performance improvements
* New analysis engines
* Dashboard enhancements
* Documentation improvements
* UI improvements
* Testing
* Workflow intelligence features

---

# Development Setup

Clone the repository:

```bash
git clone https://github.com/YOUR_USERNAME/ComfyUI-ComfyFlow-Insight.git
```

Install into:

```text
ComfyUI/custom_nodes/
```

Restart ComfyUI.

# Coding Standards

Please follow:

* Clear naming conventions
* Readable code
* Modular design
* Minimal dependencies
* Consistent formatting

---

# Pull Requests

Before submitting a pull request:

* Test your changes
* Verify functionality
* Update documentation if necessary
* Ensure no existing features are broken

---

# Feature Requests

Suggested areas:

* New workflow metrics
* Advanced dependency analysis
* Workflow comparison tools
* Visualization improvements
* Performance optimization

---

# Reporting Issues

When reporting an issue include:

* ComfyUI version
* Operating system
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots if available
